# Urban Energy CSMS — POC

מערכת ניהול עמדות טעינה (CSMS) מינימלית, מבוססת **OCPP 1.6J**.
המטרה: להוכיח שאנחנו יכולים לנהל מטען אמיתי בעצמנו (במקום הספק), לקרוא kWh
ולייצר רשומת חיוב — שאותה נחבר בהמשך ל-Priority.

## מה יש כאן
| קובץ | תפקיד |
|------|-------|
| `central_system.py` | ה-CSMS — השרת שהמטען מתחבר אליו (WebSocket, פורט 9000) |
| `db.py` | שמירת מטענים / טעינות / קריאות מונה ל-SQLite (`csms.db`) |
| `simulate_charger.py` | סימולטור מטען לבדיקה בלי חומרה |

## התקנה
```bash
pip install -r requirements.txt
```

## בדיקה מקצה לקצה (בלי חומרה)
טרמינל 1:
```bash
python central_system.py
```
טרמינל 2:
```bash
python simulate_charger.py
```
תראה ב-CSMS את כל המחזור: Boot → Authorize → Start → MeterValues → Stop,
ובסוף תהיה רשומת טעינה עם kWh בטבלת `sessions`.

## חיבור מטען אמיתי
בממשק הניהול של המטען, בהגדרת **כתובת שרת OCPP**, הזן:
```
ws://<כתובת-IP-של-המחשב-הזה>:9000/<מזהה-המטען>
```
ודא ש-OCPP מוגדר לגרסה **1.6J (JSON over WebSocket)**.
המחשב והמטען צריכים להיות באותה רשת (או חשיפת הפורט החוצה).

## למה זה ה-POC הנכון
מאשרים תאימות מול דגם המטען הספציפי לפני שמשקיעים בכל המוצר. הנקודות הרגישות:
- גרסת OCPP שהמטען תומך (1.6J מול 2.0.1)
- ה"ניב" של היצרן (שדות אופציונליים, DataTransfer ייחודי)
- אבטחה (ws מול wss/TLS, basic-auth)

## הצעדים הבאים (אחרי POC מוצלח)
1. ניהול משתמשים/תעריפים + whitelist אמיתי של כרטיסי RFID
2. דשבורד React RTL (אתרים / מטענים / טעינות)
3. מודול Priority (OData) — דחיפת רשומות חיוב, דה-דופ לפי `transaction_id`
4. טעינה חכמה (איזון עומסים), wss/TLS, זמינות 24/7
